Grill · Round 1 AI chat interface

From form to rhythm to content — six key decisions define the character of an AI chat interface.

How assistant replies appear on screen — sets the overall "conversation" vs "document" feel, and shapes long-reply readability, information density, and scroll rhythm.

Document style — long replies read comfortably; density is high; unless you're building an IM-style social product (Discord, Slack), don't force bubbles.

D.

The "still typing" signal while the assistant is streaming a reply — anchors the user's attention during the wait and shapes their perception of generation speed.

Blinking cursor — minimal footprint, matches the intuition "AI is writing", doesn't compete for attention; the most robust default.

D.

Network jitter, model timeouts and rate-limits all make a single message fail — this decides what the user sees on error, whether they have to act, and the cost / reliability trade-off.

Auto exponential backoff with N-retry cap — most failures are transient jitter; auto retry absorbs 90% of the noise, and the cap prevents infinite loops.

D.

The wordmark's font sets the brand's register at first glance — editorial / luxury / indie / modern. Body and UI fonts come later; this is the first impression.

Source Serif 4 — editorial warmth without being precious; reads as "considered, intelligent product" (Anthropic / Stripe Press lane).

D.

How conversation history is stored and passed to the model — shapes API design, token-efficiency, and how easy it is to insert / edit / branch individual messages.

Flat role/content array — the OpenAI/Anthropic native format; well-tooled; easy to insert/edit; trivial JSON serialization.

D.

What the user sees on a blank conversation — sets the AI's first impression, the user's path to first message, and the icebreaking rhythm of the chat.

Silent wait — doesn't interrupt the user's thinking; the blank canvas says "the AI is ready, your move"; especially friendly to power users who don't need a tour.

D.
